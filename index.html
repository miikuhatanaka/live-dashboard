📖 用語・KPI 定義

ダッシュボードで使っている指標の意味と計算方法

顧客リピート

リピート率2回以上購入した顧客の割合。新規獲得後の再購入力を測る指標。リピート率 = リピーター数 ÷ ユニーク顧客数 × 100%
ユニーク顧客数全期間で重複を除いた購入者数。電話番号ベースで一意に特定（無い場合は受取人名+住所）。
新規購入単価顧客の「初回注文」の平均単価。リピートした人の1回目も新規として数える。新規購入単価 = 初回注文の合計金額 ÷ 初回注文件数
リピート購入単価2回目以降の注文の平均単価。リピーターは好みが決まっているため、単価が新規と異なる傾向。リピート購入単価 = 2回目以降注文の合計金額 ÷ 2回目以降注文件数

成約系

CVR (商品別)販売数 ÷ 商品インプレッション数。その商品が表示されたうち何割が実際に売れたか。
高いほどアイコン・価格・説明が刺さっている証。CVR = sold / impressions × 100%
CVR (配信全体)購入数 ÷ アクティブ視聴者数。配信全体での購買転換率。CVR = purchases / activeViewers × 100%
CTR商品クリック数 ÷ 商品インプレッション数。TikTokが自動で出す値。商品詳細画面を開いた割合。

流通・売上

GMVGross Merchandise Value = 流通総額。税込の売上金額。手数料・原価・送料を差し引く前の数字。
販売数 (sold)注文の件数。TikTokはlisting(商品ID)単位で出すため、1個/セットなどの変種内訳は直接取れない。
本ダッシュボードではimplied unit price(GMV÷販売数)から比率を推定する加重mix計算で粗利を見積もる。

利益

粗利 (profit)売価 − 原価（仕入+梱包資材）。送料・TikTok手数料は含まない。
アフィリエイト商品は GMV × 成果報酬率 で計算される。
純利益 (netProfit)粗利から TikTok販売手数料(自社商品GMV×料率)、送料(梱包数×¥185)、梱包資材費(梱包数×¥5)を差し引いた値。
販売手数料は2026-04-20時点は一律7%（2026/5/11からカテゴリ別7〜12%へ改定予定）。
アフィリエイト商品は「販売手数料・送料・梱包資材」すべて対象外（元セラーが発送を担当・販売手数料は成果報酬5%ですでに控除済み扱い）。純利益 = 粗利 − 自社商品GMV×販売手数料率 − 梱包数×(送料+梱包資材)
unitProfit / totalProfitunitProfitは1個あたりの粗利、totalProfitはその配信での合計粗利。variant mix対象商品では加重平均。

視聴・エンゲージメント

インプレッション数LIVEや商品が表示された回数（同じ人が複数回見たらカウントされる）。
視聴数 (totalViews)視聴セッション数。1人が何度も入退室したら複数回カウント。
ユニーク視聴者 (uniqueViewers)重複を除いた実人数。
表示GPMGMV per Mille(Thousand)。 1,000インプレッションあたりのGMV。アルゴリズムの評価指標に近い。

粗利マスタ関連

masterKey粗利マスタでのSKU識別キー。例:ソックス1足, ソックス5足セット, シルクシュシュ1個。
variant mix (加重mix計算)1個バラ売り価格とセット価格の両方が存在するSKU(ソックス/メタルヘアゴム/マルチポーチ/リボン小物入れ/各シュシュ)で、implied unit price = GMV÷販売数から変種の比率を推定して加重平均で粗利を見積もる方式。
例: ソックス 7個 ¥2,211 → 単価¥315.86 → 1足¥100(4.2個) + 5足セット¥490(2.8個) と推定。
未登録 (profitMissing)粗利マスタに紐付かない商品。アフィリエイトや他社商品など、粗利計算対象外。各配信の商品明細では非表示、上の商品別サマリーでは⚠マーク付きで表示。
商品種別バッジ 🏪 自社 / 🔗 アフィリ自社: 自分で仕入れ・梱包・発送する商品。粗利から TikTok販売手数料・送料・梱包資材費が引かれる。
アフィリ(アフィリエイト): 他社商品の販売促進を代行し成果報酬を受け取る商品。元セラーが発送するため、販売手数料・送料・梱包資材は自社負担ゼロ。
現在のアフィリ商品:
  • 割れチョコ / 亀田製菓30枚セット / ハード&サワーグミ5種 — 成果報酬5%
  • 駄菓子つめあわせ25点 — 特殊アフィリ(赤字は全て自社負担、-¥200/個すべてもろもろ込み)
顧客リピート分析（※自社のみ）リピート率・新規購入単価・リピート購入単価はすべて自社商品のみで集計。
アフィリエイト商品(駄菓子・亀田・割れチョコ・グミ)は元セラーが発送するため、自社のTikTok Shop注文CSVに含まれない。そのため自然に自社商品のみの統計になる。

🌐 クラウド同期設定

GitHub GistにデータをURLするとダッシュボードをウェブ公開でき、他の人が閲覧できるようになります。
Gist ID のみ入力すると閲覧専用モード。 GitHubトークン（gistスコープ）も入力するとオーナーモード（編集・同期可能）になります。

📖 初回設定手順（クリックで展開）
【オーナー向け】
1. gist.github.com で新しいGistを作成。ファイル名を data.json、内容を [] にして「Create secret gist」。
2. GistのURLにある32文字のIDをコピー（例: https://gist.github.com/username/ここがID）。
3. github.com/settings/tokens で「gist」スコープのみのPATを作成。
4. 上のフォームに入力して「💾 設定を保存」→「🔄 今すぐ同期」でローカルデータをGistへ転送。
5. このHTMLを index.html に改名し、ファイル内の const DEFAULT_GIST_ID = '' にGist IDを貼り付けて保存。
6. GitHubリポジトリに push → Settings > Pages > mainブランチで公開 → URLをシェア。

【閲覧者向け】
オーナーからURLを受け取ったら、そのままアクセスするだけで最新データが閲覧できます。
（Gist IDが埋め込まれていない場合は、Gist IDのみ入力して「💾 設定を保存」）
